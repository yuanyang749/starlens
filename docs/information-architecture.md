# Starlens 信息架构与页面地图

> **状态：设计参考。** 页面和路由可能随实现演进；当前用户行为以 `apps/web/src/app/docs/` 和实际路由为准。

## 1. 文档目的

这份文档用于把 Starlens 当前已经确定的产品结构收束成统一的信息架构，明确：

- 公开页面与登录后页面的边界
- 核心路由与页面职责
- 页面之间的跳转关系
- 每个页面承载的主要模块

它是 `project-plan.md`、`web-ui-design.md` 和 `landing-page-design.md` 的衔接层，方便后续直接进入前端实现。

## 2. 顶层结构

Starlens 的 Web 端分为两大层：

- 公开层
  - 面向未登录用户
  - 承担产品说明、价值展示和登录承接
- 应用层
  - 面向已登录用户
  - 承担真实搜索、同步、整理、AI 和配置功能
- 移动应用层
  - 面向手机和平板用户
  - 承担小屏上的搜索、同步、整理、AI 和配置功能

对应关系：

- 公开层入口：`/`
- 应用层入口：`/app`
- 移动应用层入口：`/mobile`

## 3. 路由地图

### 3.1 公开层

- `/`
  - 落地页

### 3.2 应用层

- `/app`
  - 主工作台
- `/app/settings`
  - 应用设置总入口
- `/app/settings/ai`
  - AI Provider 配置页
- `/app/settings/tokens`
  - CLI / Agent Token 管理页
- `/mobile`
  - 手机端工作台
  - 使用底部导航承载 All Stars、Favorites、Recent 和 Settings
  - 使用 `repo` query 参数承载当前详情覆盖层

### 3.3 预留路由

以下内容在 `v1` 中可以先保持在单页内部面板或子视图实现，不强制独立路由，但在信息架构上保留扩展位：

- AI 搜索结果视图
- 收藏筛选视图
- 标签筛选视图
- 最近更新视图
- 仓库详情独立深链接

当前移动端已使用 `/mobile?repo=<id>` 作为轻量详情入口，桌面端详情深链接仍可后续扩展。

## 4. 路由状态规则

- 未登录访问 `/`
  - 正常展示落地页
- 未登录访问 `/app` 及其子路由
  - 触发登录或返回落地页
- 未登录访问 `/mobile`
  - 触发登录或返回落地页
- 移动设备访问 `/app`
  - 保留查询参数并自动跳转到 `/mobile`
- 桌面设备访问 `/mobile`
  - 自动回到 `/app`
- 已登录访问 `/`
  - 仍可看到落地页，但顶部应突出“进入工作台”
- GitHub 登录成功后
  - 默认进入 `/app`

## 5. 页面地图

### 5.1 `/` 落地页

页面职责：

- 介绍 Starlens 是什么
- 展示真实产品形态
- 强调核心价值与能力边界
- 承接 GitHub 登录
- 连接开源与自部署信息

主要模块：

- 顶部导航
  - 产品名
  - GitHub 仓库入口
  - 登录按钮 / 进入工作台按钮
- 首屏 Hero
  - 标题与副文案
  - 主 CTA
  - 工作台预览图
- 核心功能区
  - 搜索与过滤
  - 标签、备注、收藏
  - AI 辅助
- 工作方式区
  - Web
  - CLI
  - Agent
- 协议与 AI 支持区
  - Gateway
  - OpenAI-compatible
  - Anthropic
  - Gemini
- 开源与自部署区
- 页脚

### 5.2 `/app` 主工作台

页面职责：

- 承载用户真实的日常使用流程
- 在一个稳定工作台里完成搜索、查看、整理和 AI 辅助

主要模块：

- 顶部栏
  - 全局搜索框
  - 手动同步
  - AI 搜索入口
  - 快捷过滤
  - 用户菜单
- 左侧导航
  - All Stars
  - Favorites
  - Recent
  - By Language
  - By Tag
  - AI Search
  - Tokens
  - Settings
- 中间结果区
  - 仓库列表
  - 排序与过滤状态
  - 空状态 / 无结果状态
- 右侧详情区
  - 仓库详情
  - 标签编辑
  - 备注编辑
  - 收藏切换
  - AI 总结

### 5.3 `/app/settings` 设置总入口

页面职责：

- 作为设置域的容器页和导航入口
- 聚合 AI、Token 以及后续扩展配置项

主要模块：

- 设置导航
- 设置概览卡片
- 跳转到 AI 配置和 Token 管理

### 5.4 `/app/settings/ai` AI Provider 配置页

页面职责：

- 管理用户自己的 AI provider 配置
- 新增、编辑、启用、停用、校验和设置默认 provider

主要模块：

- Provider 列表
- 新增 provider 表单入口
- 编辑表单 / 抽屉
- 默认 provider 切换
- 校验状态展示
- 模型列表读取入口

### 5.5 `/app/settings/tokens` Token 管理页

页面职责：

- 管理 CLI / agent 的访问 token
- 提供创建、撤销和查看使用状态的入口

主要模块：

- Token 列表
- 新建 token
- 最近使用时间
- 撤销操作
- 权限与说明提示

### 5.6 `/mobile` 手机端工作台

页面职责：

- 在小屏上承载核心日常使用流程
- 用单列列表、详情覆盖层和底部导航替代桌面三栏工作台
- 保持与桌面端一致的数据能力和 API 合同

主要模块：

- 顶部区域
  - 产品名和当前用户
  - 搜索框
  - AI 搜索按钮
  - 同步按钮
  - 语言和标签过滤
- 内容区
  - 仓库列表
  - 收藏状态
  - 滚动加载
  - 空状态和错误状态
- 详情覆盖层
  - 仓库摘要
  - Star / Fork / Language 信息
  - 标签编辑
  - 备注编辑
  - 收藏切换
  - GitHub 和 Homepage 外链
- 设置视图
  - General
  - Providers
  - Tokens
- 底部导航
  - Stars
  - Favorites
  - Recent
  - Settings

## 6. 核心用户路径

### 6.1 首次使用路径

1. 访问 `/`
2. 阅读产品说明与核心能力
3. 点击 GitHub 登录
4. 登录成功后进入 `/app`
5. 首次同步 starred repos
6. 进入工作台开始搜索和整理

### 6.2 日常使用路径

1. 进入 `/app`
2. 在顶部搜索或左侧切换过滤视图
3. 在结果区浏览仓库
4. 在右侧详情中查看摘要、备注、标签和 AI 总结
5. 原地完成整理动作

### 6.2.1 手机端日常使用路径

1. 在移动设备访问 `/app` 或 `/mobile`
2. 系统进入 `/mobile`
3. 在顶部搜索、同步或过滤仓库
4. 在单列列表中浏览结果
5. 点开仓库详情覆盖层
6. 原地完成收藏、备注和标签编辑

### 6.3 AI 配置路径

1. 从工作台用户菜单或侧栏进入设置
2. 打开 `/app/settings/ai`
3. 新增或编辑 provider
4. 校验连接
5. 设置默认 provider
6. 返回工作台使用 AI 功能

### 6.4 Agent / CLI 接入路径

1. 进入 `/app/settings/tokens`
2. 创建 token
3. 在 CLI 或 agent 中配置 token
4. 使用统一 API 访问搜索、同步和 AI 能力

## 7. 页面之间的导航关系

主要导航关系如下：

- `/`
  - 可跳转到 GitHub 登录
  - 可跳转到 GitHub 仓库
  - 已登录时可跳转到 `/app`
- `/app`
  - 可跳转到 `/app/settings`
  - 可跳转到 `/app/settings/ai`
  - 可跳转到 `/app/settings/tokens`
  - 移动设备会自动跳转到 `/mobile`
- `/app/settings`
  - 可跳转到 `/app/settings/ai`
  - 可跳转到 `/app/settings/tokens`
- `/mobile`
  - 桌面设备会自动回到 `/app`
  - 可通过底部导航切换工作台视图
  - 可通过 `repo` query 打开或关闭详情覆盖层

## 8. 信息层级原则

### 公开层优先级

1. Starlens 是什么
2. 为什么值得用
3. 如何开始使用
4. 它还能扩展到哪些能力

### 应用层优先级

1. 找到仓库
2. 理解仓库
3. 整理仓库
4. 配置 AI 和自动化入口

## 9. 当前默认决策

- 落地页与工作台分路由，不混在同一页面
- 工作台是核心使用界面，不承担产品介绍职责
- 设置域作为 `/app` 下的子路由存在
- AI 配置和 Token 管理优先作为独立设置页面，而不是嵌在主工作台弹窗里
- `v1` 不强制给每个筛选视图做独立路由，先以工作台内切换为主
- 移动端优先保证核心工作台闭环，不追求桌面布局等比复刻
- 移动端设置入口先放在 `/mobile` 内部视图，避免小屏路由和导航层级过深
