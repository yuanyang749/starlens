# Starlens 落地页设计文档

## 1. 页面定位

Starlens 的落地页是公开可访问的产品入口页，路由为 `/`。

它的职责不是营销式宣传，也不是一个空壳登录门面，而是：

- 向首次访问者快速说明 Starlens 是什么
- 展示真实产品形态和使用价值
- 通过短演示视频证明核心使用闭环
- 承接 GitHub 登录入口
- 对外说明 CLI、agent、开源、自部署和多 AI provider 支持

与之对应，登录后的真实工作台路由为 `/app`。

## 2. 目标受众

落地页同时面向两类受众：

- 个人开发者用户
  - GitHub stars 很多，希望快速找回、整理、补充备注
- 开源与技术受众
  - 关心 CLI、agent 集成、自部署和 AI provider 兼容能力

文案优先顺序：

1. 个人使用价值
2. 技术能力说明
3. 开源与扩展能力

## 3. 整体设计方向

落地页的视觉语言必须和工作台属于同一个产品体系。

核心气质：

- 安静专业
- 克制现代
- 工具感强
- 不花哨
- 不营销化

设计要求：

- 不做典型 SaaS 营销页套路
- 不用夸张 hero 插画
- 不做强烈渐变、炫光、紫色 AI 风
- 不堆砌卖点卡片和空泛口号
- 不把落地页做成宣传片首页

## 4. 路由与状态规则

- `/`
  - 公开落地页
- `/app`
  - 登录后工作台

默认行为：

- 未登录访问 `/`：正常看到落地页
- 未登录访问 `/app`：跳转登录或回到 `/`
- 已登录访问 `/`：仍可查看落地页，但顶部应突出“进入工作台”
- GitHub 登录成功后：默认进入 `/app`

## 5. 首屏设计

首屏采用**分屏并重**布局。

### 左侧内容

- 产品名：`Starlens`
- 一句主标题，明确它是 GitHub Stars 的检索与整理工作台
- 一段短副文案，强调：
  - 快速找回 starred repos
  - 给仓库补充标签、备注和收藏状态
  - 用 AI 辅助理解和筛选结果
- 主按钮：`使用 GitHub 登录`
- 次按钮：`查看 GitHub 仓库`

### 右侧内容

- 展示真实的工作台预览图，作为首屏主视觉
- 预览图来自 Starlens 自身的产品界面风格
- 使用 21:9 工作台 mockup 作为主视觉
- 可在同一区域加入一条辅助性的 Remotion 短演示视频

### Remotion 视频策略

- 首屏视频是辅助媒体，不替代主截图
- 默认行为为：静音、自动播放、循环
- 首屏主短片建议时长：`10-15 秒`
- 首屏视频内容以“搜索到整理”的完整闭环为主：
  - 输入搜索
  - 浏览结果
  - 选中仓库
  - 打标签、写备注或收藏
- 不使用夸张镜头语言，不做宣传片式动画包装

要求：

- 首屏不应只有文字，也不应只有截图
- 文字和产品预览需要同时成立
- CTA 必须在首屏直接可见
- 首屏第一视觉仍应被识别为“真实产品”，而不是“视频宣传内容”

## 6. 页面内容结构

### 6.1 首屏

目标：

- 让用户第一眼知道这是做什么的
- 让用户快速进入登录使用
- 让用户直接看到产品真实使用方式

### 6.2 核心功能区

建议展示三组能力：

- 搜索与过滤
  - 快速找回你 star 过的仓库
- 整理与沉淀
  - 标签、备注、收藏、摘要
- AI 辅助
  - 问答、候选重排、仓库总结

要求：

- 用结构化短文案说明
- 每块能力都应配简短 UI 片段或局部预览
- 避免空泛价值词堆砌
- 页面中后段应允许放置更短的演示视频，而不是只靠静态说明

### 6.3 工作方式区

说明 Starlens 不只是网页：

- Web 工作台
- CLI
- Agent 接入

需要明确表达：

- 同一套数据和能力可以被不同入口复用
- 适合人直接用，也适合自动化工作流

补充媒体策略：

- 这里可加入一条 `5-8 秒` 的 Remotion 微演示
- 内容聚焦 CLI / agent 接入，不与首屏主短片重复

### 6.4 AI 与协议支持区

说明 AI 兼容能力：

- Vercel AI Gateway
- OpenAI-compatible
- Anthropic 官方协议
- Gemini 官方协议

表达重点：

- 不锁死单一 AI 平台
- 用户可以自己选择 provider
- AI 是搜索增强层，不替代主检索链路

补充媒体策略：

- 这里可加入一条 `5-8 秒` 的 Remotion 微演示
- 内容聚焦 AI 问答、候选重排或仓库摘要
- 视频只证明 AI 的使用方式，不夸大 AI 的主导地位

### 6.5 开源与自部署区

说明项目面向开源与扩展：

- 支持自部署
- 支持 PostgreSQL
- 支持 Vercel 起步部署
- 适合作为个人工具，也适合作为开源项目演进

### 6.6 页脚

包含：

- GitHub 仓库链接
- 登录入口
- 简短版权或项目状态说明

## 7. 未登录用户体验

`v1` 不提供公开可交互 demo。

未登录用户能看到：

- 静态产品预览
- 静音自动循环的短演示视频
- 功能说明
- 技术与开源能力说明

未登录用户不能做：

- 搜索真实仓库数据
- 进入工作台
- 试用同步功能
- 使用真实 AI 检索链路

## 8. 视觉规范

### 8.1 色彩

沿用工作台风格：

- 白、浅灰、深灰、近黑为主
- 单一冷色强调
- 不使用高饱和背景

### 8.2 排版

- 字体现代、中性、克制
- 标题不应过大
- 页面整体更偏产品说明，而非广告海报

### 8.3 组件

- 小圆角
- 轻边框
- 极轻阴影
- 按钮克制，不做夸张色块

### 8.4 动效

- 仅保留必要的进入和 hover 反馈
- 不做花哨滚动动画
- 不用夸张的分屏飞入或 3D 动效
- Remotion 视频的节奏应稳定、简洁、可信，像产品演示而不是品牌片

## 8.5 视频规范

如果使用 Remotion，默认规范如下：

- 首屏：一条主短片
- 中段：两条补充微演示
- 组织方式：
  - 一条主短片证明核心闭环
  - 一条 AI 微演示
  - 一条 CLI / Agent 微演示
- 所有视频都基于真实 UI 视觉语言制作
- 不允许把视频作为整屏背景
- 不允许用视频替代 CTA、标题和产品截图的角色

## 9. 与工作台的关系

落地页和工作台必须共享同一套品牌与视觉语言，但职责明确分离：

- 落地页负责说明和承接
- 工作台负责真实使用

明确约束：

- 不把营销内容塞进 `/app`
- 不把真实工作台变成演示页
- 不让落地页和工作台看起来像两个不同产品
- 不让 Remotion 视频看起来像另一个独立品牌资产

## 10. 验收标准

- 未登录用户能在首屏清楚理解 Starlens 是什么
- GitHub 登录 CTA 明确且优先级足够高
- 首屏产品预览与后续真实工作台风格一致
- 首屏加入视频后，页面主感受仍然是产品工具而不是宣传页
- 页面既能打动个人用户，也能回应开源技术受众
- CLI、agent、AI provider、开源能力说明准确，不夸大未实现能力
- 落地页整体克制、专业，不落入营销模板风格

## 11. 当前默认决策

- 路由：`/` 为落地页，`/app` 为工作台
- 首屏：分屏布局，左文案右产品预览，允许一条辅助性短视频
- 受众：个人用户与开源受众兼顾
- 内容重点：功能说明 + CLI/agent + 开源与 AI 协议支持
- 未登录体验：展示静态预览与静音短视频，不提供交互 demo
- 视频策略：一条主短片 + 两条补充微演示
